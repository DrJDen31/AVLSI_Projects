# 1. MATLAB Design & Code Structure

## Filter Design Approach
The objective of this project was to design a 100-tap low-pass FIR filter with a transition band of $0.2\pi \sim 0.23\pi$ rad/sample and a stopband attenuation of at least 80 dB. 

Our initial pass using the Parks-McClellan optimal equiripple algorithm (`firpm` in MATLAB, `scipy.signal.remez` in Python) revealed that **100 taps is insufficient** to meet the 80 dB attenuation requirement given the steep transition band. An automated iterative loop determined that a minimum of **175 taps** is required to meet the specification.

The final filter was designed using these parameters:
- **N (Taps)**: 175
- **Passband Edge**: $0.2\pi$
- **Stopband Edge**: $0.23\pi$
- **Weighting**: Ratio of stopband ripple target to passband ripple target to aggressively emphasize the 80 dB stopband.

## Verification Stimulus Generation
To verify the Verilog implementations, test vectors were generated mathematically and saved as hex files:
1. **Impulse**: Verifies the raw coefficients propagate correctly through the delay line and accumulators.
2. **DC Signal**: Checks maximum accumulator growth without frequency dynamics.
3. **Passband Tone**: Ensures signals within $0.2\pi$ pass unaltered.
4. **Stopband Tone**: Ensures signals above $0.23\pi$ are heavily attenuated.
5. **Composite Signal**: Tests a mix of passband, stopband, and random noise to simulate real-world ADC input.

The golden outputs were generated using bit-exact integer convolution of the quantized input vectors and the quantized coefficients.

## Verilog Code Structure
The hardware repository is structured to separate functionality and maintain a clean test bench environment:

- **`coeff_pkg.sv`**: A central SystemVerilog package auto-generated by the design script. It contains the quantized coefficient array, number of taps, coefficient width, data width, and the calculated safe accumulator width. All FIR modules import this package.
- **Top-level Modules (e.g., `fir_direct.sv`, `fir_parallel_L2.sv`)**: Each architecture is instantiated as its own standalone top-level module with identical `valid_in` / `valid_out` AXI-Stream-style handshake interfaces.
- **`fir_tb.sv`**: A universal testbench that uses `$readmemh` to load the test vectors and automatically checks the DUT output against the golden reference, allowing for a configurable ±1 LSB tolerance for rounding truncation differences.
