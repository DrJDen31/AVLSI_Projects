\begin{abstract}
This report details the implementation of seven distinct FIR filter architectures, ranging from a direct-form baseline to a high-speed Pipelined Fast FIR (FFA) design. We analyze performance trade-offs in terms of area (ALMs), power, and maximum frequency ($F_{max}$), demonstrating that the FFA L3 architecture provides an optimal balance for parallel DSP processing.
\end{abstract}

\section{Introduction}
Digital Finite Impulse Response (FIR) filters are essential components in modern communication systems. As tap counts increase—in this case 175 taps—the hardware complexity and timing constraints become significant challenges. This project explores parallelism and pipelining to overcome these bottlenecks.

\section{Architectures}
\subsection{Direct Form (L=1)}
The baseline architecture utilizing a single multiply-accumulate (MAC) chain.

\subsection{Pipelined Direct Form}
Enhanced $F_{max}$ by transforming the serial summation into a \textbf{Pipelined Adder Tree}, reducing the critical path to 175/PIPE\_EVERY levels.

\subsection{Parallel Architectures (L=2, L=3)}
Standard polyphase implementation where $L$ samples are processed per clock cycle.

\subsection{Fast FIR Algorithm (L=2, L=3)}
Reduced-complexity parallel designs using the Fast FIR Algorithm (FFA) to minimize the number of required multipliers.
